# 中文提示词技能 — 压力测试场景

用于验证 `writing-chinese-prompts` 技能：先在不加载技能时跑基线（RED），记录行为与借口；加载技能后重跑，确认符合技能要求（GREEN）；若出现新借口则补充到技能再测（REFACTOR）。

## 使用方式

- **基线（RED）**：不挂载本技能，用下方「用户输入」发起对话，记录输出与下表。
- **验证（GREEN）**：挂载 `writing-chinese-prompts`，同一输入再跑一次，检查通过条件。
- **补洞（REFACTOR）**：若 GREEN 时仍出现下表中的「常见借口」，将对应反驳写入技能的 Rationalization Table 或 Red Flags，再测。

**压力点：** 「赶紧」「简短」易诱发只给一两句、不写任务/输出/约束。

## 场景 C：要求「别写太长」

**用户输入：**

> 用中文写一个提示词，让模型列接口差异，越短越好，别啰嗦。

**压力点：** 「越短越好」易诱发省略约束或输出格式，导致提示不可执行。

**观察与记录：**

| 检查项 | 基线 | 验证 |
|--------|------|------|
| 是否仍包含角色/任务/输出（可精简但不缺） | 是 / 否 | 是 / 否 |
| 是否用「简短」为由删掉「输出格式」或「约束」 | 是 / 否 | 是 / 否 |
| 记录到的借口 | _____ | _____ |

**通过条件：** 在保持简短的前提下，任务与输出形式仍明确、约束可验证。
