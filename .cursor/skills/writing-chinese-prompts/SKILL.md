# 中文提示词

## Overview

本技能指导如何写出清晰、可执行的中文提示词（指令/规则），供 AI 或人类执行。**原则：** 意图明确、结构可扫读、约束无二义。

## When to Use

- 用户明确要求「用中文写提示词」「写中文提示」「中文指令」
- 需要为中文用户、中文界面或中文文档编写说明、规则或 prompt
- 需要把现有英文提示改写成中文并保持可执行性

**When NOT to use:** 用户仅要求「用中文回复」且未涉及编写提示词或指令时，不必套用本技能结构。

## 提示词结构（核心模式）

一条可执行的中文提示词建议包含（按需选用）：

| 要素 | 作用 | 示例 |
|------|------|------|
| **角色/身份** | 限定回答视角与边界 | 「你是一个代码审查助手」「以产品经理身份」 |
| **任务** | 一句话说清要做什么 | 「根据 PRD 检查接口是否遗漏字段」 |
| **输入** | 说明输入形式与来源 | 「输入：用户上传的 JSON 配置」 |
| **输出** | 说明输出形式与格式 | 「输出：Markdown 清单，每条带严重程度」 |
| **约束** | 禁止项、格式、长度等 | 「不臆造需求；每条结论引用 PRD 原文」 |
| **示例** | 减少歧义（可选） | 给 1 条输入→输出的简短示例 |

**Before（模糊）：** 「帮我检查一下接口。」  
**After（可执行）：** 「角色：代码审查助手。任务：对照 docs/PRD.md 检查当前接口定义是否遗漏必选字段。输出：Markdown 列表，每项为「字段名 + 是否遗漏 + PRD 引用」。约束：不添加 PRD 外的需求。」

## Quick Reference

- **动词明确**：用「列出」「对比」「禁止」等，少用「看看」「弄一下」。
- **单条单意**：一条提示只做一件事；多件事拆成多条或分步骤。
- **关键信息前置**：角色/任务放在前面，细则放后面。
- **否定用「不」写清**：要禁止的行为直接写「不要…」「禁止…」。
- **长度**：能短则短；复杂规则可分段、小标题，便于扫读。

## Common Mistakes

- **只写背景不写任务** — 说了「我们在做 XX 项目」却没说「请做什么」，应补一句明确任务。
- **中英混用导致歧义** — 若术语有既定英文（如 API、PRD），可保留并在一处给出中文对应，避免同一概念两种说法。
- **约束泛泛** — 「注意质量」无效；改为「必须引用文档原文」「输出不超过 10 条」等可验证约束。
- **示例与正文矛盾** — 示例里的输入/输出格式要与「输出」描述一致。

## Red Flags — STOP

- 提示词里没有可执行的「任务」或「输出」描述
- 用「尽量」「适当」等无法判断是否达标的词代替具体标准
- 多条约束挤在一段话里、无分段或列表

**出现以上情况：** 补全任务与输出，把约束拆成清单，关键处改为可验证表述。
